# CHANGELOG

История версий приложения Grizzly Messenger.

## Workflow деплоя

**ВАЖНО:** перед каждым деплоем клиента (`./deploy.sh onlyClient`) обязательно:

1. Поднять `versionCode` и `versionName` в [app/build.gradle.kts](app/build.gradle.kts)
2. Добавить новую запись наверх этого файла по шаблону ниже
3. Запустить `./deploy.sh onlyClient` — скрипт прочитает changelog последней
   версии из этого файла и зарегистрирует её в БД через `POST /admin/versions`,
   после чего загрузит APK на сервер

После деплоя версия и changelog становятся доступны:
- В админ-панели: вкладка **Версии** (https://grizzly-messenger.ru/admin/)
- В клиенте: эндпоинт `GET /v1/app/update` отдаёт самую свежую версию
- В диалоге обновления приложения у пользователей

## Шаблон записи

Каждая версия — заголовок второго уровня в формате
`## [имяВерсии] (кодВерсии) — ДАТА`, дальше пустая строка и булет-список изменений.
Парсер `deploy.sh` берёт **первую** такую запись сверху файла как актуальную версию.
Содержимое code-блоков (\`\`\`) игнорируется, используйте их для примеров.

Про админскую часть ничего не пишем

Не забываем менять версию в build.gradle.kts

## [1.0.65 alfa] (65) — 2026-04-14

- Пароль шифрования: при входе на новом устройстве приложение предлагает ввести пароль для расшифровки переписки. Один пароль — один ключ на любом устройстве. Минимум 8 символов, буквы + цифры
- Если пароль забыт — можно удалить ключ шифрования (с подтверждением через SMS) и задать новый. Старые сообщения при этом будут безвозвратно потеряны
- Можно пропустить ввод пароля при регистрации и задать его позже в настройках
- Маска номера телефона при авторизации: формат (###) ###-##-## для РФ, свой формат для каждой страны. Кнопка «Продолжить» активна только при полном номере
- Иконка «глаз» в полях ввода пароля — показать/скрыть символы
- Управление сессиями: в настройках раздел «Активные сессии» — список всех входов с устройством, IP, геолокацией (страна, город). Можно завершить любую сессию или все сразу кроме текущей
- При завершении сессии другое устройство мгновенно выбрасывает на экран авторизации
- При смене пароля шифрования все остальные устройства автоматически разлогиниваются
- Нерасшифрованные сообщения скрыты из списка и превью чатов вместо плашки «Не удалось расшифровать»
- Разблокировка старых сообщений в настройках: ввод пароля → переподгрузка и расшифровка
- Бот Grizzly Bot: рассылки от администратора отображаются как обычные входящие сообщения. В чате с ботом нельзя писать и звонить
- Исправлены пустые аватарки у пользователей, чьи аватары были удалены — теперь показывается placeholder

## [1.0.64 alfa] (64) — 2026-04-12

- Полный редизайн экрана звонка: размытый фон с аватаром собеседника, пульсирующие волны вокруг аватара во время вызова, «стеклянная» панель управления, таймер длительности соединения, индикатор сквозного шифрования
- Гудок исходящего вызова с паузой 3 секунды между повторами
- Звук успешной отправки сообщения (текст / голос / картинка)
- Во время аудио-звонка экран автоматически гаснет когда телефон поднесён к уху (датчик приближения), и включается обратно когда убираете. Если устройство не поддерживает системное гашение — экран накрывается чёрным оверлеем, тачи блокируются
- Время дозвона увеличено до 5 минут (раньше 30 секунд) — звонок больше не завершается сам пока собеседник тянется к телефону
- Запись голосового сообщения по удержанию пальца на иконке микрофона: вокруг кнопки идут красивые расходящиеся волны, на месте поля ввода — живая waveform-волна, реагирующая на громкость голоса. Отпустили — открывается превью с плеером и кнопками «Отправить» / «Отменить»
- Загрузка аватара профиля: после выбора фото из галереи или камеры открывается полноэкранный кропер с круглой маской — pinch-to-zoom двумя пальцами, перетаскивание одним. Получается ровный квадратный аватар без обрезанных лиц
- Когда собеседник меняет аватар или имя — у вас в списке чатов и в карточке профиля всё обновляется сразу через WebSocket, без перезахода в приложение
- Картинки, которые не удалось загрузить, теперь показываются как нейтральная плашка-заглушка с иконкой, а не красным текстом ошибки
- Удаление чата теперь действительно удаляет: при перезапуске приложения или синхронизации с сервером скрытый чат не возвращается. Если собеседник снова напишет — чат появится автоматически
- Исправлен «не в сети» статус собеседника в списке чатов: индикатор больше не висит серым у тех, кто реально онлайн. При переподключении WebSocket список чатов и статусы пересинхронизируются автоматически
- При запуске приложения список чатов и шапка профиля показываются как «дышащие» серые скелетон-плейсхолдеры вместо мелькающего пустого состояния
- Пропущенный звонок: у звонящего теперь «Звонок без ответа», у получателя — «Пропущенный звонок» (раньше у обоих было одинаково). У звонящего больше не висит +1 непрочитанных за свой же неудачный вызов
- Авторизация: лимит 3 попытки на ввод OTP-кода. После 3 неудач код инвалидируется, надо запросить новый
- Все ошибки авторизации теперь на русском с понятным текстом: «Неверный код. Осталось попыток: 2», «Срок действия кода истёк. Запросите новый», «Слишком много неверных попыток. Запросите новый код»
