# CHANGELOG

История версий приложения Grizzly Messenger.

## Workflow деплоя

**ВАЖНО:** перед каждым деплоем клиента (`./deploy.sh onlyClient`) обязательно:

1. Поднять `versionCode` и `versionName` в [app/build.gradle.kts](app/build.gradle.kts)
2. Добавить новую запись наверх этого файла по шаблону ниже
3. Запустить `./deploy.sh onlyClient` — скрипт прочитает changelog последней
   версии из этого файла и зарегистрирует её в БД через `POST /admin/versions`,
   после чего загрузит APK на сервер

После деплоя версия и changelog становятся доступны:
- В админ-панели: вкладка **Версии** (https://grizzly-messenger.ru/admin/)
- В клиенте: эндпоинт `GET /v1/app/update` отдаёт самую свежую версию
- В диалоге обновления приложения у пользователей

## Шаблон записи

Каждая версия — заголовок второго уровня в формате
`## [имяВерсии] (кодВерсии) — ДАТА`, дальше пустая строка и булет-список изменений.
Парсер `deploy.sh` берёт **первую** такую запись сверху файла как актуальную версию.
Содержимое code-блоков (\`\`\`) игнорируется, используйте их для примеров.

Про админскую часть ничего не пишем

Не забываем менять версию в build.gradle.kts

## [1.0.69 alfa] (69) — 2026-04-25

- Группы: аватары теперь можно загружать. В создании группы рядом с полем «Название» появился круглый плейсхолдер - тап открывает галерею с кропером; в «Информации о группе» CREATOR/ADMIN видит маленькую кнопку-камеру в углу аватара для смены позже
- Группы: ADMIN тоже может назначать новых администраторов из обычных участников. Раньше это мог только создатель - теперь админы могут расширять команду модерации сами
- Группы: передача владельца группы. Создатель через меню «Передать владельца» отдаёт роль другому участнику, сам становится администратором. В чат прилетает системное сообщение, новые роли тут же видны у всех
- Профиль и группы: тап по аватару разворачивает фото inline до половины экрана с заездом поверх системного бара (часы и батарея остаются видны поверх фото).
- Звонки: ваше превью с фронтальной камеры теперь зеркальное. У собеседника изображение приходит в естественной ориентации - помашете правой рукой, у него она тоже будет правой
- Звонки: качельки громкости теперь регулируют канал «голосовой вызов» (а не «звонок») - поднятие громкости делает собеседника слышнее, а не увеличивает рингтон будущих звонков
- Видеозвонки: адаптация под качество сети. Разрешение по умолчанию 960×540 при 24 кадрах вместо 1280×720@30 - визуально на телефоне разница незаметна, а нестабильному интернету намного проще тянуть. Битрейт видео ограничен 800 kbps - алгоритм congestion control больше не «душит» аудио из-за гонки за качеством картинки. При просадке сети предпочитаем сохранять плавность движения (frame rate) ценой временной размытости — лучше плавный диалог, чем красивые «застывшие» кадры
- Система и уведомления: при первом запуске чек-лист критичных разрешений (уведомления, поверх блокировки экрана, фоновая работа) с кнопками-ярлыками прямо в системные настройки. Если случайно отказались - можно «Не напоминать», и вернуться позже через настройки
- Система: иконка «Настройки» в нижней панели подсвечивается красной пульсирующей точкой, если не хватает критичных разрешений - чтобы вы не пропустили
- Система: отдельный экран «Диагностика разрешений» в настройках с подробным статусом каждого разрешения и сведениями об устройстве (модель, Android, бренд) - удобно показать, если что-то не работает
- Система: на устройствах Xiaomi/Huawei/Samsung/Vivo/OPPO/Honor/OnePlus/ASUS добавлен отдельный пункт «Автозапуск (для производителя)» - у них своя система убийства фона сверх стандартной Android, без неё звонки часто не доходят
- Группы: фикс - после полного выхода из приложения и повторного входа собственные сообщения в группе не расшифровывались (показывалась плашка «не удалось расшифровать»). Теперь свой sender-ключ всегда сохраняется и для самого себя - после login всё на месте
- Группы: расшифровка стала надёжнее в целом - клиент пробует все имеющиеся sender-ключи отправителя по всем epoch'ам, а не только последний. Старые сообщения после нескольких ротаций перестали превращаться в «не удалось расшифровать»
- Группы: после смены состава (добавили/исключили кого-то) sender-ключи автоматически рассылаются всем оставшимся, новый участник сразу получает свой ключ для отправки
- Группы: меню действий по участнику (3 точки) теперь всплывает прямо из-под нажатой иконки. Если у вашей роли нет доступных действий по этому участнику (например ADMIN на ADMIN'е) - меню не появляется, чтобы не показывать пустое
- Шапка группы «Информации о группе» приведена к стилю профиля - крупный заголовок 30sp с адаптивным размером под ширину экрана, не съезжает на маленьких устройствах
- О группе «Добавить участника» в карточке участников теперь по центру строки
- Модалка «Добавить участника» переделана: аватары рядом с именами, поле поиска (по имени, юзернейму, телефону), под именем — @username или номер телефона. Когда у вас в адресной книге много зарегистрированных контактов — нужный находится за секунду
- Группы: фикс — приглашённый участник теперь видит новую группу мгновенно. Раньше после создания группы у приглашённых она не появлялась, пока они не сделают новый syncChats (мог не происходить часами); теперь сервер шлёт каждому приглашённому WS-событие group_member_added в момент создания
- Группы: фикс — FCM-уведомления о новых сообщениях. Если получатель оффлайн, и сообщение пришло через WebSocket-канал (а не REST), теперь тоже отправляется push на устройство (раньше push шёл только из REST-пути; в результате часть групповых сообщений приходила «молча»)
- Группы: возможность приглушить уведомления группы. В «Информации о группе» переключатель «Уведомления»: пуши приходят, но шторку клиент не показывает (мьют локальный, без ведома сервера — собеседник не узнаёт). Состояние сохраняется и работает даже когда приложение убито
- Группы: создатель может удалить группу полностью. В «Информации о группе» кнопка «Удалить группу» (видна только CREATOR'у) с подтверждением. Сервер каскадом удаляет переписку и ключи у всех участников, всем приходит WS-событие group_deleted и группа исчезает из списка чатов

## [1.0.68 alfa] (68) — 2026-04-24

- Групповые чаты: первый релиз. Создание через меню FAB «Новая группа» в списке чатов - задаёте название, выбираете контакты (до 49 человек + вы сами), жмёте «Создать»
- Групповые чаты: вся переписка сквозно шифруется (Sender Keys, Signal-подобная схема) - сервер хранит только зашифрованные блобы, никто кроме участников не может прочитать сообщения
- Групповые чаты: текст, голосовые и картинки шифруются и доставляются точно так же как в личных чатах - никаких отличий по UX
- Групповые чаты: роли CREATOR / ADMIN / MEMBER. Создатель может назначать и снимать администраторов, администраторы - добавлять и исключать участников
- Групповые чаты: экран «Информация о группе» открывается тапом по аватару или названию в шапке - список участников с ролями, добавить, исключить, переименовать, выйти из группы
- Групповые чаты: при выходе создателя права владельца автоматически передаются случайному администратору (если есть) или случайному участнику - группа никогда не остаётся «без хозяина». Если уходит последний участник - группа удаляется целиком
- Групповые чаты: при смене состава (добавление/исключение) sender-ключи автоматически ротируются - кикнутый больше не может расшифровать новые сообщения
- В списке чатов у группы в правом нижнем углу аватара появилась маленькая иконка с двумя силуэтами - теперь видно с первого взгляда, что это группа
- Над чужими сообщениями в группе показывается имя автора - больше не нужно гадать, кто написал
- В шапке группы теперь видно сколько всего участников и сколько из них онлайн прямо сейчас. Если кто-то печатает - пишем имя; если несколько - «X и Y печатают», от трёх - «X и ещё N печатают»
- Системные уведомления о составе группы прямо в чате: «X добавил(а) Y», «X исключил(а) Y», «X покинул(а) группу», «X стал(а) владельцем группы», «X назначил(а) Y администратором» - выглядят как центрированные плашки между сообщениями (стиль уведомлений о звонках)
- Защита от участников на старой версии: если в группе кто-то ещё не обновил приложение и не сможет читать сообщения, показываем замок у его имени в «Информации о группе» и красную плашку под шапкой чата с просьбой попросить обновиться
- Меню действий по участнику (3 точки) теперь всплывает прямо из-под нажатой иконки, а не где-то в начале списка
- Ограничение 1.0.68: аватары групп пока нельзя загружать - отображается буква-заглушка по первому символу названия; загрузка аватара появится чуть позже
- Ограничение 1.0.68: новые участники видят только сообщения, полученные после добавления - прошлая история для них недоступна, расшаривание истории появится чуть позже
- Обратная совместимость: прямые (direct) чаты полностью не затронуты, direct-переписка и звонки работают как раньше; старые клиенты 1.0.67 продолжают работать с сервером 1.0.68

## [1.0.67 alfa] (67) — 2026-04-24

- Экран авторизации: при активной клавиатуре шапка с логотипом плавно сжимается, а форма с полями и кнопкой полностью видна - кнопка «Продолжить» больше не уезжает под клавиатуру на маленьких экранах
- Экран авторизации: добавлен корректный отступ сверху с учётом выреза фронтальной камеры (hole-punch/notch) - логотип и название больше не заезжают под иконку камеры
- Экран авторизации: все поля ввода (страна, номер, OTP, пароль, повтор пароля) теперь одинаковой высоты на любом устройстве - раньше на некоторых телефонах они были разного размера
- Диалог: крупный рефакторинг списка на reverseLayout - при открытии чата пользователь видит последние сообщения МГНОВЕННО, без промежуточной фазы «загружаем всё сверху, потом скроллим вниз». Новые сообщения и своя отправка автоматически прокручиваются к низу, история вверху - через обычный скролл
- Диалог: индикатор подгрузки старых сообщений теперь занимает стабильную область в 40 dp сверху списка — его появление/исчезновение больше не «дёргает» сообщения ниже
- Диалог: дата-разделители стали полноценными элементами списка (а не рисовались инлайново) - это исключает скачки при обновлениях и корректно работает в reverseLayout
- Диалог: подгрузка старых сообщений триггерится за 5 айтемов до верхнего края - дозагрузка идёт невидимо, без ощущения «ожидания»
- Стикеры: повышен лимит для больших стикеров (до 1.2 МБ вместо 600 КБ) - теперь отправляются анимированные стикеры из крупных паков Gboard/Telegram
- Большие статичные стикеры (PNG/WebP с прозрачностью и размером > 768 px) теперь корректно ужимаются с сохранением альфа-канала, а не превращаются в «фото на белом фоне»
- Если стикер всё равно слишком большой - показывается понятное сообщение об ошибке («Стикер слишком большой X МБ, максимум Y МБ») вместо тихого проглатывания
- Онлайн-статус: исправлен баг, когда собеседник после долгого перерыва заходил и всё равно показывался «не в сети». WebSocket-клиент теперь помнит намерение по presence и повторно отправляет его серверу сразу после установки соединения
- Онлайн-статус: при открытии чата клиент дополнительно подтягивает актуальный профиль собеседника через REST - даже если WS-событие status было пропущено, статус обновится мгновенно
- Настройки: при смене темы со светлой на тёмную (или обратно) играется настоящий pixel-perfect circular reveal - из центра экрана разрастается «окно» с новым интерфейсом, снаружи остаётся старый, между ними чёткая граница. Снимок старого UI делается через классический Android-подход `View.draw(Canvas)` в битмап - под кругом виден РЕАЛЬНЫЙ новый интерфейс, а не сплошная заливка цветом.
- Настройки: внутри одной «светлоты» (два тёмных варианта или два светлых) reveal не проигрывается - там достаточно плавного color crossfade, круг выглядел бы как лишнее движение. Повторный тап по уже выбранной схеме ничего не триггерит
- Плавное переключение тем: все цвета интерфейса (фон, пузырьки, шапка, инпут, иконки) переливаются при смене схемы - переход с тёмной на светлую больше не бьёт по глазам резкой вспышкой (ps: Флешку уже не словите 😀)
- Android 10/11: graceful degradation для эффекта матового стекла в чате. Блюр (BlurEffect) доступен только с Android 12 - на более ранних версиях шапка и поле ввода теперь рендерятся с повышенной непрозрачностью (0.96/0.97 вместо 0.72/0.88), чтобы сообщения за ними не просвечивали и экран оставался чётким
- В настройках (раздел «Цветовая схема») для пользователей Android < 12 теперь показывается информационная плашка о том, что матовое стекло доступно только на Android 12+. На Android 12+ плашки нет - не засоряем интерфейс

- Нерасшифрованные сообщения скрыты из списка и превью чатов вместо плашки «Не удалось расшифровать»

- Бот Grizzly Bot: рассылки от администратора отображаются как обычные входящие сообщения. В чате с ботом нельзя писать и звонить
