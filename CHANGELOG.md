# CHANGELOG

История версий приложения Grizzly Messenger.

## Workflow деплоя

**ВАЖНО:** перед каждым деплоем клиента (`./deploy.sh onlyClient`) обязательно:

1. Поднять `versionCode` и `versionName` в [app/build.gradle.kts](app/build.gradle.kts)
2. Добавить новую запись наверх этого файла по шаблону ниже
3. Запустить `./deploy.sh onlyClient` — скрипт прочитает changelog последней
   версии из этого файла и зарегистрирует её в БД через `POST /admin/versions`,
   после чего загрузит APK на сервер

После деплоя версия и changelog становятся доступны:
- В админ-панели: вкладка **Версии** (https://grizzly-messenger.ru/admin/)
- В клиенте: эндпоинт `GET /v1/app/update` отдаёт самую свежую версию
- В диалоге обновления приложения у пользователей

## Шаблон записи

Каждая версия — заголовок второго уровня в формате
`## [имяВерсии] (кодВерсии) — ДАТА`, дальше пустая строка и булет-список изменений.
Парсер `deploy.sh` берёт **первую** такую запись сверху файла как актуальную версию.
Содержимое code-блоков (\`\`\`) игнорируется, используйте их для примеров.

Про админскую часть ничего не пишем

Не забываем менять версию в build.gradle.kts
